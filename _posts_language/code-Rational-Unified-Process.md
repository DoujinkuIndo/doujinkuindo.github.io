## Rational Unified Process
래쇼날 유니파이드 프로세스(Rational Unified process, 합리적 통합 프로세스)라고 하는 객체지향 방법론을 발표.

1. 요구분석
   1. 시스템 정의
   2. 시스템 정의 상세
2. 분석과 설계
   1. 아키텍처 분석
   2. 행위 분석
   3. 아키텍처 상세
   4. 상세 설계
3. 구현
   1. 컴포넌트 설계
   2. 설계 요소 구현
   3. 단위 테스트

<details><summary>요구 분석</summary>
<div markdown="1">

요구분석(requirement) 작업흐름(workflow)의 목적은 시스템이 무엇을 해야 하는지에 대한 고객과 사용자의 의견을 일치시키고, 시스템 개발자가 시스템의 요구사항을 잘 이해할 수 있게 하며, 시스템의 범위를 결정하고 제약 사항을 파악하기 위한 것 입니다.

</div></details>

<details><summary>시스템 정의</summary>
<div markdown="1">

프로젝트에서 가장 처음 해야 할 일은 무엇을 구축할 것인가를 명확하게 정의하는 것 입니다. 시스템의 범위를 결정하는 것을 말합니다. 어디가 시스템의 결계이며, 들어오는 것은 무엇이고 무엇이 나가는 지를 정의해야 합니다. 

* 이 과정에서 액터(actor)와 유스케이스(use case)를 찾아내어 유스케이스 모델(use-case model)에 정의합니다. 아직 이 단계에서 유스케이스를 세부적으로 기술할 필요는 없습니다.
* 용어와 개념을 개발 구성원들이 서로 다르게 이해하는데서 오는 혼란을 피하기 위해 공통 용어와 개념을 일치시켜 용어집(glossary)에 기록합니다. 일단 시스템의 범위가 결정되면 찾아낸 유스케이스의 우선 순위를 결정합니다.

**액터와 유스케이스 찾기**

액터(actor)란 구현하고자 하는 시스템과 서로 작용하는 사람이나 다른 시스템을 말합니다. 따라서 액터는 시스템의 일부분이 아니며, 시스템 외부에 있는 것 입니다. 그러믈 액터를 찾는 것은 시스템의 범위를 결정하는 것과 상관이 있습니다.
