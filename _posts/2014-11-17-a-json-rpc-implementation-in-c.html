Recently I developed an rpc library in C++ based on json formatted communication. The code is on github.

This implementation is just another toy project I did due to boring engineer life at bay area and it doesn't guarantee any stability for your significant and serious work. But if you happen to be interested in rpc protocol and wonder how it works, you may want to check it out and play with it.

My json-rpc will need you to write an spec file, in json format, to generate a stub file, which needs you to fill up the functions, or shall we call it services, and then compile to binary. You need to specify the name of your service, the serializable classes and the services. After you finish the spec file, you could use that little program under bin/ to generate the stub file. You can find more details on github.

Then the only thing you need to do is fill up those functions you want the server to support. Please notice the rpc implementation itself doesn't protect your functions in multiply threaded mode. If there is some critical area you need to protect, you have to do that on your own in generated service class.

Anyway, have fun with json-rpc.
