# URLconf

URLconf其实就是起地址定向（或叫配对）的作用，浏览器中输入的不同地址定向到不同的处理逻辑上。

例如，下面定义了网址根目录下的定向方法：
{% highlight python %}
urlpatterns = [
    url(r'^polls/', include('polls.urls', namespace='polls')),
    url(r'^admin/', include(admin.site.urls)),
]
{% endhighlight %}

而下面的代码又定义了`polls.urls`里的二级目录定向方法：
{% highlight python %}
urlpatterns = [
    url(r'^$', views.IndexView.as_view(), name='index'),
    url(r'^login/$', views.login, name='login'), ##
    url(r'^register/$', views.register, name='register'),
    url(r'^logout/$', views.logout, name='logout'),
]
{% endhighlight %}

所以，像`127.0.0.1:8000/polls/login/`就会由指定的视图函数`views.login`来处理。

# 模型（Model)

一个模型对应着一个数据库中的表，所以也可以说模型就是数据。

例如，下面定义了一个Django模型：
{% highlight python %}
class Question(models.Model):
    question_text = models.CharField(max_length=200)
    pub_date = models.DateTimeField('date published')
{% endhighlight %}
它对应的数据表如下：
{% highlight sql %}
CREATE TABLE "polls_question" (
    "id" serial NOT NULL PRIMARY KEY,
    "question_text" varchar(200) NOT NULL,
    "pub_date" timestamp with time zone NOT NULL
);
{% endhighlight %}

# 视图（View）

真正处理浏览器请求的是视图。视图接收请求数据，调用其他模块产生结果数据，然后再把结果数据传给模板，由模板最终产生返回给用户的HTML页面。

例如，下面是一个简单的注册视图：
{% highlight python %}
def register(request):
    if request.method == 'POST':
        form = UserCreationForm(request.POST)
        if form.is_valid():
            new_user = form.save()
            return HttpResponseRedirect("/polls")
    else:
        form = UserCreationForm()
    return render(request, "polls/register.html", {'form': form})
{% endhighlight %}
