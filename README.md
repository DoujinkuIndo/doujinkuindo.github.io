### HowToWork
- 잘 모르는 것은 지레짐작하지 않고, 있는 그대로만 말합니다.
  - 잘 모르기 때문에 틀릴가능성이 높기 때문입니다.
- 급하게 대답하지 말고, 한번더 생각한 다음에 대답하도록 합니다.
- 프로그래밍을 잘한다의 기준
  - 구조를 잘 작성한다.
  - 로직, 함수를 잘 작성한다.
  - API, 프레임워크등을 잘 활용한다.
- 사람마다 다를순 있지만, 코드에 오류나 경고가 없어야 합니다.
  - 사람이 완벽할 순 없으므로, 오류나 경고가 있다면 일단 물어보고 작업합니다.
- 같이 일할 땐, 냄새가 나지 않아야 합니다... 이것은 예의입니다.
  - 어떻게 홀애비 냄새가 나지 않는가.
