연결 리스트(Linked List)는 데이터를 저장하는 선형 데이터 구조로, 각각의 데이터 요소가 노드(Node)로 구성되어 있습니다. 각 노드는 데이터 자체와 다음 노드를 가리키는 링크(또는 포인터)로 이루어져 있어 데이터 간의 연결이 형성됩니다.

연결 리스트의 중요한 특징은 다음과 같습니다:

* 동적 크기: 연결 리스트는 동적으로 크기를 조절할 수 있어서 데이터를 필요할 때마다 추가하거나 삭제할 수 있습니다. 크기를 미리 정하지 않고도 데이터를 유연하게 다룰 수 있습니다.
* 랜덤 접근 어려움: 배열과는 달리 연결 리스트는 특정 위치의 데이터에 직접 접근하는 것이 어렵습니다. 데이터를 찾으려면 처음부터 순차적으로 노드를 따라가야 합니다.
* 중복 허용과 순서 유지: 연결 리스트는 중복된 데이터를 허용하며, 데이터가 삽입된 순서대로 유지됩니다.
* 데이터 추가 및 삭제 효율적: 연결 리스트는 데이터의 추가 및 삭제 연산을 효율적으로 처리할 수 있습니다. 특히 맨 앞이나 맨 뒤에서 데이터를 추가 또는 삭제하는 경우에는 빠른 시간 복잡도를 가집니다.
* 메모리 효율성: 연결 리스트는 데이터를 노드로 분할하여 저장하기 때문에 메모리를 효율적으로 활용할 수 있습니다.

연결 리스트의 종류로는 단일 연결 리스트(Singly Linked List), 이중 연결 리스트(Doubly Linked List), 원형 연결 리스트(Circular Linked List) 등이 있으며, 각각의 종류에 따라 노드 간의 연결 방식과 사용 용도가 다를 수 있습니다. 연결 리스트는 데이터를 동적으로 관리하고 삽입/삭제를 효율적으로 처리해야 할 때 주로 활용됩니다.
