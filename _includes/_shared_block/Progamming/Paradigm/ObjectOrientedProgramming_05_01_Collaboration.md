객체지향에서 협력(Collaboration)은 객체 간의 상호작용을 의미합니다. 객체들은 서로의 존재를 인식하고, 서로 연결되어 메시지를 주고받으며 협력을 수행합니다. 협력은 객체지향에서 핵심적인 개념 중 하나이며, 객체 지향 시스템의 동작을 결정합니다.

객체지향에서 객체는 다른 객체와 상호작용하여 문제를 해결하고, 목적을 달성하기 위해 협력합니다. 이때, 객체는 다른 객체에게 메시지를 전송하고, 메시지를 수신한 객체는 해당 메시지를 처리하여 적절한 결과를 반환합니다. 객체 간의 협력은 객체가 수행하는 역할과 책임에 따라 결정되며, 객체들은 서로 협력하여 목적을 달성합니다.

예를 들어, 은행 시스템에서 계좌 객체는 입금, 출금, 잔액 조회 등의 기능을 제공합니다. 고객 객체는 계좌 객체에게 입금, 출금을 요청하며, 계좌 객체는 요청을 처리하여 결과를 반환합니다. 이때, 고객 객체와 계좌 객체는 서로 협력하여 은행 시스템의 동작을 결정합니다.

객체지향에서는 객체 간의 협력을 중요시하며, 객체들은 적절한 협력을 통해 목적을 달성합니다. 협력은 객체지향 시스템의 동작을 결정하는 핵심적인 개념 중 하나이며, 객체지향 설계에서 객체들의 책임과 역할을 정의할 때 고려해야 합니다. 따라서 **협력이 설계를 위한 문맥을 결정합니다.** 애플리케이션 안에 어떤 객체가 필요하다면 그 이유는 단 하나여야 합니다. 그 객체가 어떤 협력에 참여하고 있어야 합니다. 그리고 객체가 협력에 참여할 수 있기 위해서는 협력에 필요한 **적절한 행동**을 보유하고 있어야 합니다.

* 애플리케이션의 기능을 구현하기 위해 어떤 협력이 필요하고 협력을 위해 어떤 역할과 책임이 필요한지를 고민하지 않은 채 너무 이른 시기에 구현에 초점을 맞추면, 변경하기 어렵고 유연하지 못한 코드를 만드는 원인이 됩니다.
